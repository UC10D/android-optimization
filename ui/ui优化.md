# UI性能优化

* ui渲染流程
  * cpu(计算)=>draw call=>gpu(绘制)
* ui问题的表现    
  * 感觉不到问题, 或者稍微卡顿
* 解决方案
  * 减少Overdraw过渡绘制
  * 优化layout, 减少cpu对节点的计算

### Overdraw的检测

* 数据收集: 手机检测: 设置 -> 开发者选项 -> 调试GPU过度绘制 -> 显示GPU过度绘制   
* 分析问题

```
Overdraw

1  <  2  <   3 < 4

蓝  < 绿  <  粉 <  红

```    

* 解决办法
  * 移除不必要的background(xml, code)
  * 使用clipRect: 自定义view

```
getWindow().setBackgroundDrawable(null);

# 去除
android:background="@android:color/black"

```
